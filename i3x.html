FAQ

i3X, answered.

What is i3X?

i3X (Industrial Information Interoperability Exchange) turns disparate data sources — SQL databases, MTConnect devices, OPC-UA, and CSV/Excel files — into standardized, queryable REST APIs with semantic context models, so any application can simply ask for the data it needs.

Which data sources does i3X support?

Relational databases (MSSQL, PostgreSQL, MySQL, SQLite, Snowflake, DB2), MTConnect devices via MQTT, and CSV/Excel files with auto-detected cross-file relationships. You can also aggregate multiple i3X servers into one endpoint, and an OPC-UA bridge (i3xopc) is on the way.

Do I have to write integration code?

No. i3X performs automatic schema discovery — introspecting tables, columns, keys, and constraints — with one-click auto-mapping and a visual canvas. Foreign keys become composition hierarchies, and columns become queryable attributes with live values, quality, and timestamps.

What’s in the i3X suite?

GUI tools — i3xdb (SQL mapper), i3xmt (MTConnect), i3xcsv (flat files), i3xview (node-graph visualizer), i3xdash (live dashboards), and i3X Explorer — plus headless Docker/CLI servers for each, and i3xrag, an MCP server for AI.

Can AI agents use i3X?

Yes. i3xrag is an MCP server that lets AI agents run full-text (BM25) search, traverse the data graph, find paths, and analyze the model. It works with Claude Desktop and other compatible clients.

How is i3X deployed and configured?

Flexibly: embedded servers inside the desktop apps, standalone headless CLI servers for Docker and CI pipelines, and browser-based viewers. Configuration is YAML-based, so it version-controls cleanly.

Does it serve real-time data or just static schemas?

Real-time. i3X serves live values with quality and timestamp metadata, and includes live dashboards with value, gauge, sparkline, and timeline tiles.
